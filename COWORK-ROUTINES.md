# Cowork / Claude Code 定期タスク仕様（情報源の発掘・審査・順位付け + 嗜好レポート）

このファイルは、デイリーブリーフの「判断が要る・低頻度・クリティカルパス外」の
仕事を、週1の定期タスク(Cowork または Claude Code の Routine)に任せるための
**仕様書 兼 プロンプト全文**です。ユーザーはこの中の「週次Routineプロンプト」を
自分の定期タスクに貼って設置します。SYSTEM-DESIGN §7・§8 の設計をそのまま実装
したもの。

---

## どこで動かすか

- **コワーク / コード どちらでも動く**(両方 Anthropic のクラウドで定期実行が可能)。
- 推奨: **Claude Code の定期タスク**。理由は my-brain が git リポジトリで、
  clone→編集→commit→push がネイティブで確実、Supabase も Bash の curl で読める
  (コネクタ設定不要)。コワークで動かす場合は GitHub コネクタの接続が要る。
- スケジュール: **週1・オフ時間(例: 日曜 03:00 JST)**。クリティカルパス外。
  走らない週があっても、夜間Cron(Gemini)は現状の sources.md をそのまま巡回し
  続けるので、ブリーフは止まらない(graceful degradation)。

### ★Claude Code(コード)で動かす時の必須設定2つ(2026-07-20 実機で判明)

1. **環境のネットワークアクセスを「Full」にする**。ルーティン編集画面の環境
   セレクタ(☁)→「環境を追加」→**ネットワークアクセス=Full**(無制限)で環境を
   作り、その環境をこのルーティンに割り当てる。既定の「デフォルト」環境は
   **Trusted**(npm等パッケージ取得のみ許可・一般Webは403でブロック)なので、
   Trustedのままだと WebFetch/curl が全ホスト403になり発掘できない
   (`example.com`すら弾かれる=サイト側でなく出口ポリシーの403。curl小細工でも
   越えられない)。Full は「なし/Trusted/Full/カスタム」の選択肢の一つ。
2. **デフォルトブランチへ直接コミットさせる**(プロンプトの「# コミット」節)。
   コードは既定で新しい `claude/...` ブランチへpushしPR待ちで止まる=デフォルト
   ブランチに入らずアプリに反映されない。プロンプトで直コミットを明示する。
   それでもブランチが作られたら、GitHubで「Compare & pull request」→Merge で
   取り込む(または設定→Claude Code「プルリクエストを自動的に作成する」+GitHub
   auto-merge)。

## このタスクが持つ責務

日次のブリーフ生成(巡回→抽出→分類→編成)はアプリ側 Gemini(Vercel Cron)が
毎日やる。このタスクはそこに**乗せる情報源の質と並び**、および**嗜好の
振り返り**だけを持つ。

1. **情報源の発掘・審査**(§3-4/§3-7): 好み・興味に合う質の高いサイトを探し、
   審査し、合格分を情報源プールへ足す。
2. **情報源プールの順位付け・淘汰**(§7-4): KEEP実績で情報源を並べ替え、
   上限(〜40)を超えたら実績の低いものを落とす。お気に入り(ユーザー登録)は
   淘汰しない。
3. **嗜好の差分レポート**(§7-6/§8): KEEP/SKIPログを分析し、好み・語り口・
   深さの傾向と変化を人間可読レポートにして書き戻す(Obsidianで読む)。
   taste-state.md の好み・興味は書き換えない(確定はユーザー本人)。

## 必要なアクセス・環境変数(タスク側に設定)

- my-brain リポジトリの git 読み書き(本人のリポジトリ)。
- Web 検索・閲覧(発掘・審査用)。
- Supabase(**読み取りのみ**): `SUPABASE_URL` / `SUPABASE_SERVICE_ROLE_KEY` /
  `OWNER_USER_ID`。

## 扱うファイル(my-brain)

- `taste-state.md`(**読むだけ**): 「## 好み」「## 興味」「## 生活圏」「## 願い」。
- `sources.md`: 情報源リスト。**アプリ管理の
  `<!-- BEGIN app-managed:favorites -->`〜`<!-- END app-managed:favorites -->`
  ブロックには絶対に触れない**(お気に入り=ユーザー登録・淘汰対象外)。
  このタスクが管理するのは下記 Cowork ブロックだけ:
  ```
  <!-- BEGIN cowork:discovered -->
  ## 発掘した情報源（KEEP実績順）
  - [サイト名](https://example.com) — 一行の採用理由 ｜ 実績: KEEP 3 / SKIP 1
  <!-- END cowork:discovered -->
  ```
  アプリの夜間Cronは sources.md 内の全箇条書きURLを読むので、このブロックの
  URLも自動で巡回対象になる(アプリ改修不要)。
- `cowork/rejected.md`: 審査で不合格にしたサイトのURLと理由(再審査回避)。
- `taste-report.md`: 嗜好の差分レポート(このタスクが毎回まるごと書き直す)。

## ★コワークにはGitHubコネクタが無い場合の半自動版（2026-07-20 実機で判明）

コワーク(コネクタ方式)には GitHub コネクタが用意されていないことがある
(実機で確認: Gmail/Calendar/Drive/Spotify/TickTick のみ、GitHub検索でも出ない)。
その場合コワークからは my-brain に直接書けない。**git書き込みを諦め、コワークは
「発掘して一覧で出す」だけにして、登録はアプリの「設定 → お気に入りの情報源」
から人手で行う**(この欄は元々 my-brain の sources.md へ同期され夜間生成に使われる)。
GitHub連携ゼロで今すぐ回せる。フル自動(git直書き)がしたい場合は、コネクタでなく
git ネイティブな **Claude Code(コード)側でタスクを作り、my-brain をリポジトリに
追加**する(下の「どこで動かすか」参照)。

### 半自動版プロンプト全文（コワーク・GitHub不要・出力を人手で登録）

```
# 役割
あなたはデイリーブリーフというアプリの情報源スカウトです。ユーザーが関心を持つテーマについて、質の高い書き手による記事を継続的に発信しているWebサイトを探し、記事を実際に読んで審査し、合格したサイトを一覧で出力します。ファイルへの書き込みはしません（ユーザーがアプリに手で登録します）。

# 前提（この下にユーザーが自分の値を書いておく）
好み: 建築, 器・工芸, 西洋美術
興味: 西洋美術の展覧会, ミニシアターの単館上映
生活圏: 東京23区
既に登録済みで重複させないサイト: （あれば列挙）

# 探す
「好み」「興味」の各テーマを検索語にして、それらを扱う記事・サイトをWeb検索する。見つけた良いサイトや記事が本文中でリンク・言及している別のサイトもたどる。特定のサイト名を検索語にして「それに似たサイト」を探すことはしない。

# 審査する
運営者が個人か組織かは合否に関係しない。サイトの規模や更新頻度でなく、文章そのもので判断する。候補サイトごとに、記事を2〜3本（できれば別々のテーマ）と、サイトの紹介ページ（"About"・「このサイトについて」・「運営者情報」など運営方針や書き手の立場が書かれた場所）を実際に開いて読む。
次の4つをすべて満たすサイトだけを合格とする:
1. 事実の紹介・要約だけで終わらず、書き手自身の評価・解釈・主張が含まれている。
2. プレスリリース・Wikipedia・他サイトの言い換えでない、その書き手独自の情報や観察が含まれている。
3. 固有名・年代・背景・経緯に具体的に踏み込んでおり、その分野に精通した人が書いたと判断できる情報量がある。
4. 紹介ページや記事全体から、一貫した関心・方針・視点が読み取れる。
次のいずれかに当てはまるサイトは不合格とする: 他サイトからの寄せ集め・まとめ・ランキングが中心で書き手自身の文章が乏しい / アフィリエイトリンクや「おすすめ◯選」形式が主な内容 / 検索流入狙いのキーワードの繰り返しが目立つ / プレスリリースやニュースをほぼ転載している / 全文がペイウォール内・ログイン必須・本文がJavaScriptでしか表示されない。
判断に迷った場合は不合格とする。

# 出力
合格したサイトを最大5件、1件ずつ「サイト名 / URL / 合格と判断した具体的な理由（どの記事のどこがどう良かったか）」の形式で出す。続けて、不合格にした候補を「URL — 一行の不合格理由」で挙げる。

# 守ること
- 実際に開いて本文を読んだサイトだけを出す。URLを推測で作らない。
```

登録手順: アプリの **設定 → お気に入りの情報源** に、出力されたURLを1件ずつ貼って追加する(この欄が my-brain の sources.md へ同期され、夜間生成の巡回対象になる)。

## 段階を分けて設置する

# 仕事2: プールの順位付け・淘汰(cowork:discoveredブロックだけを書き換える)
1. cowork:discovered ブロックの各情報源について、利用ログからそのドメイン(カードの sourceUrl のホスト)由来のカードの KEEP数・SKIP数・旗数 を集計する。
2. 情報源を次の優先度で並べ替える(上ほど、夜間Cronに早く巡回される):
   (a) 実績のある情報源を KEEP率が高い順に上へ。旗が付いた情報源は下げる。
